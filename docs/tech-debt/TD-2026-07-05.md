# TD-2026-07-05: Metrics exporter smoke test

**Source:** Review session 01, Round 1 — pr-test-analyzer #7 (MEDIUM).
**Status:** open

## Problem

`init_metrics` (PrometheusBuilder install + HTTP listener) is now called at
startup (session 01) but has zero runtime test coverage; a
metrics-exporter-prometheus behavior change would be discovered at
production startup. The unit tests only verify the `record_*` helpers don't
panic without a recorder installed.

## Binding trigger

At the next metrics-exporter-prometheus major/minor bump, add one test that
calls `init_metrics` on an ephemeral port, records a counter, and fetches
`/metrics` asserting the metric name appears.
